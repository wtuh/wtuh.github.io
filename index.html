利用github搭建有专属域名的免费，简单，快捷的个人博客

Posted on 2024-04-04

如果爱折腾探索且需要一个专属免费的个人网页，这篇看完将帮助你！作者也将自己付费域名nyada.cn的二级域名(如a.nyada.cn, a是自定义域名区)拿出10个送给志同道合的朋友，联系方式进nyada.cn左侧有邮箱。

一、正式的搭建之前，我们需要有这样的思路。

  1. 在哪里展示我们的博客呢？经过我一年来的了解github pages是一个非常好的选择，这就需要你有一个github账号（如注册账号为：github.com/用户名），那么你的个人主页项目搭建后将存在于名为用户名.github.io的项目中，后续你可以通过用户名.github.io访问你的个人主页。

  2. 如果你想有一个自己的专属自定义的域名，你需要去域名注册网站（阿里云、腾讯云等等）然后解析到用户名.github.io即可实现有专属域名的个人博客。

  3. 怎么运营管理我们的博客呢？有一些利用WordPress等等建立的博客，在网页后台可以随时随地的管理。这种是不错，但对于个人来说，我自身就喜欢在本地进行管理然后git同步到个人主页上。最近我神奇的发现了这样的本地管理工具——Hexo，他是简洁、强大且稳定的。进一步的你想美化主页也可以用官方的主题。

二、搭建原理和优缺点

4. Hexo + GitHub 文章发布原理

在本地撰写 Markdown 格式文章后，通过 Hexo 解析文档，渲染生成具有主题样式的 HTML 静态网页，再推送到 GitHub 上完成博文的发布。

5. 优点和不足

优点：完全免费；静态站点，轻量快速；可按需求自由定制改造；托管在 GitHub，安全省心；迁移方便…

不足：发文不便，依赖于本地环境；更适合个人博客使用；GitHub 在国内访问速度有点不快。

三、Next 主题和其他资源

6. 使用 Next 主题美化个人博客

Hexo 默认的 Landscape 主题不够美观，换成流行的 Next 主题吧！通过对 Next 主题简单的设置，基本可以满足新手所有的需求。

7. 其他资源

除 Next 之外，Hexo 还有非常丰富的主题，可以浏览 Hexo 官网寻找适合自己口味的主题。

随着博客数量和网站需求的增加，可能需要多种多样的扩展功能，可以在 Hexo 官网插件页面找到满足自己需求的插件。

四、绑定域名（可选）

这里也可以免费使用作者的二级域名形如a.nyada.cn（a为自定义域名区，限定前十个联系的人）。博客搭建完成使用的是 GitHub 的子域名（如用户名.github.io），我们可以为 Hexo 博客绑定自己的域名替换 GitHub 域名，更加个性化和专业，也利于 SEO。

8. 域名注册和解析

利用域名注册网站，如阿里云、腾讯云等。然后在 DNS 设置部分，删除自带的记录，然后添加 CNAME 记录将 www 域名解析指向：用户名.github.io。

9. 绑定域名到 Hexo 博客

进入本地博客文件夹的 source 目录，打开记事本，里面输入自己的域名，如 http://a.nyada.cn，保存名称为 “CNAME”，格式为 “所有文件”（无 .txt 后缀）。

10. 开启 HTTPS

配置自己的域名后，需要我们手动开启 HTTPS。打开博客所在 GitHub 仓库，Settings -> 下拉找到 GitHub Pages -> 勾选 Enforce HTTPS。

end

知乎crystal总结说：

Hexo 是一种纯静态的博客，我们必须要在本地完成文章的编辑再部署到 GitHub 上，依赖于本地环境。不能像 WordPress 或 Typecho 那样的动态博客一样能直接在浏览器中完成撰文和发布。

可以说是一种比较极客的写博客方式，但是优势也是明显的——免费稳定省心，比较适合爱折腾研究的用户，或者没有在线发文需求的朋友。

categories

Posted on 2024-04-02

单词解释

“categories” 是英文中的一个词，通常用于描述事物或概念的分类或类别。在计算机领域，尤其是在网站或博客的开发中，”categories” 通常指代文章或内容所属的类别或分类。在网站或博客的内容管理系统中，可以使用 “categories” 来将文章或内容进行分类，以便读者更容易地找到感兴趣的内容。在中文中，”categories” 可以翻译为 “分类” 或 “类别”。

categories和tags的区别

“categories”（分类）和”tags”（标签）在内容管理系统中用于对文章或内容进行分类和标记，但它们在功能和用途上有一些区别：

  1. 分类（Categories）：
    • 分类通常用于对文章或内容进行更广泛的主题性分类。它们代表了文章所属的主要类别或主题。例如，一个博客可能有 “科技”、”健康”、”旅行” 等分类。
    • 一篇文章通常只能属于一个分类，它代表了文章的主要内容或主题。
  2. 标签（Tags）：
    • 标签则是用于对文章或内容进行更具体的描述性标记。标签通常用于描述文章的关键词、主题或内容特点，使读者能够更容易地找到相关内容。例如，对于一篇关于健康的文章，可能会有 “健康”、”饮食”、”运动” 等标签。
    • 一篇文章可以有多个标签，这样可以更细致地描述文章的内容特点。

因此，分类更多地用于对文章进行宏观的整体分类，而标签则更适合用于对文章进行细致的描述和关联。

如何在网站中设置categories

在网站中设置文章分类（categories）通常是通过网站的内容管理系统（CMS）或使用特定的网站建设平台来完成的。下面是一般的步骤：

  1. 登录后台管理界面： 使用你的管理员账户登录到你的网站后台管理界面。
  2. 找到分类设置选项： 寻找一个用于管理文章分类的选项或页面。这通常在网站管理界面的“文章”或“内容”部分。
  3. 创建分类： 一般情况下，你会看到一个可以添加、编辑或删除文章分类的界面。在这里，你可以创建新的分类，例如：“科技”、“健康”、“旅行”等。
  4. 指定文章分类： 在撰写或编辑文章时，你应该能够从已创建的分类中选择一个适当的分类。这通常在撰写文章的编辑界面中的一个下拉菜单或复选框中完成。
  5. 保存更改： 选择好文章分类后，确保保存你的文章或更改。这样，文章就会被归类到你所选择的分类中。

具体的步骤可能会因网站建设平台或CMS系统而异，但通常都遵循类似的流程。如果你使用的是特定的网站建设平台（如WordPress、Wix、Squarespace等），可以参考相关平台的帮助文档或社区支持资源，了解更详细的指导。
