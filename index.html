  • 基因调控网络是基因和基因之间的相互作用，生化反应网络你准备研究什么物质？
  • 生物反应如果研究RNA分子，那本质和基因比较接近，他们之间有什么联系呢？
  • 深度神经网络本质上基因调控网络有很大区别的，能不能实现用深度神经网络去训练一个基因调控网络呢？

推进计划怎么去写？

Posted on 2024-04-16 | In b

项目推进计划的重要组成部分

可以总结出项目推进计划的几个重要组成部分，大家在制作项目推进计划时可以对照一下，以免遗漏重要信息。

  1. 项目背景和目标

项目推进计划的第一部分应该清楚地介绍项目的背景信息和目标。这包括项目的名称、描述、背景、关键利益相关方，以及项目的主要目标和预期成果。项目背景和目标的明确表述有助于团队对项目的整体理解，并确保所有参与者在同一方向上努力前进。

  1. 项目范围

这一部分要详细说明项目的范围，包括项目的边界、可交付成果和排除项。明确定义项目的范围有助于避免范围蔓延和范围变更，确保项目保持焦点并按计划推进。

  1. 时间管理

时间管理部分包括项目的时间表和进度计划。它显示了项目任务的排列顺序、起始和完成日期，以及任务之间的依赖关系。我们通常会使用甘特图或项目网络图等工具可视化项目的时间线。时间管理是项目推进计划中的关键部分，有助于确保项目按时交付。

  1. 资源管理

这一部分涉及项目所需的资源，包括人员、设备、材料和技术。它描述了如何分配和管理这些资源，以满足项目需求。资源冲突解决策略也可能在此部分进行讨论。

  1. 风险管理

风险管理部分识别了项目可能面临的风险和机会，并提供了相应的风险应对策略。这包括风险评估、风险缓解计划和应急计划，以确保项目能够应对各种不确定性。

  1. 沟通计划

沟通计划部分定义了项目团队和利益相关方之间的沟通渠道、频率和内容。它确保信息流畅地传递，团队成员和利益相关方了解项目的进展和重要决策。

  1. 监控和控制

监控和控制部分涉及如何监控项目的进展，识别问题并采取纠正措施。它包括定义关键绩效指标（KPIs）、报告进展、执行变更管理和问题解决流程。

  1. 关键绩效指标（KPIs）

项目的成功与否通常由关键绩效指标来衡量。这些指标应与项目目标和范围一致，并用于评估项目的整体表现。KPIs 可以包括进度、质量、成本、客户满意度等方面的度量标准。

  1. 附录和附加信息

最后，项目推进计划可能包括附录和附加信息，如详细的任务清单、资源分配表、工作分解结构（WBS）等。这些附加信息有助于提供更全面的项目管理视图。

研究计划research proposal

研究计划的创新点

很多同学在找我们团队辅导research proposal的时候都提到需要一个新的Idea，别人没有报道过的。我每次遇到这些同学都会耐心的给他们讲一个观点：我觉得对于理工科同学而言，创新的idea是指你在前人研究基础上的创新，别人研究还有什么不完整或者需要改进的地方；对于文科同学而言，研究视角的创新是最为重要的一种创新。

千万不要认为别人没有报道就是创新，根据我的经验别人没有报道的东西大概率是没有研究价值的东西，一个遗落的珍珠被你捡到的可能性太低，不要把宝贵的青春用在幻想的舞台，有这个时间还不如多谈一场恋爱。

简而言之，研究问题的创新，研究方法的改进和研究视觉的新颖都是创新。无论硕士还是本科生申请博士，想要找到一个新的研究问题，难度都太大。所以建议大家在后面两种创新上下功夫。

举个例子：之前有个辅导研究计划的同学问我“老师，我觉得你给我提炼的这个课题已经有类似的研究了？我回答他：“研究过的东西仍然是有研究价值的，你研究的视角不同，体系不同，方法不同，都是创新，都可以是你博士期间的研究计划。比如：研究马克思主义对中国影响的人很多，为什么还不停有人去研究呢？这个也从侧面说明，研究的人越多往往说明了研究价值越高。

研究计划的一般框架

研究计划一般由Title，Background，Literature Review，Research Gap，Research Question，Research Aim， Research Method，Expected Results and Time Plan等几个部分组成。当然各个学校要求的肯定不一样，但是大同小异，目的都是讲清楚研究计划。我们在写作研究的时候都是先按照上面的步骤确定思路和框架然后再进行写作。因此，一个逻辑严密的思路和合理的框架是写好Research Proposal全文的关键。最后，需要提醒大家注意一个问题，研究计划还应该和自己目标导师的研究相关，所以在写研究计划之前多读读目标导师发表的论文是必要的。下图有助于大家理解研究计划的框架及其关系：

飞书甘特图怎么用

来自：https://www.zhihu.com/question/360821332/answer/1527127192

甘特图一般包含三个要素：任务、资源（人力）、时间，换成大白话来説，就是什么人在什么时间内完成什么任务。

基于此，甘特图的绘制原则一般有以下3条：

1）任务具体可执行：每项任务必须是具体的可执行的工作，最好有明确的优先级；

2）任务分配合理：人力资源一定要和工作内容匹配，让合适的人做合适的事；

3）时间安排合理：时间期限不能由管理层拍脑门敲定，具体的执行人员一定要参与时间的评估，要给任务预留缓冲时间。

简要介绍下背景

UNIX:中文名尤尼斯，本质是操作系统，属于分时操作系统另一种（时时操作系统）开发于1969年，AT&T公司，特点多任务、多用户。unix是一个强大的多用户、多任务操作系统，支持多种处理器架构。不开源

Linux：创始人 林纳斯·托瓦兹，属于操作系统内核，内核加软件才是操作系统。发型版本Debian、Ubuntu、REHEL等，特点 是免费、开源、可靠、安全、稳定、多平台。

Linux全程GNU/Linux，一种免费使用和自有传播的类Unix的操作系统，其内核由林纳斯·本纳迪克特·托瓦兹于1991首次发布，主要受到Minix和Unix思想的启发。基础POSIX的多用户、多多任务、支持多线程和多cpu的操作系统。

GNU：是一个操作系统，其内容软件完全以GPL方式发布。这个操作系统是GNU计划的主要目标，名称来自GNU‘s Not UNIx！的递归缩写，因为GNU的设计类似Unix，但它不包含具著作权的Unix代码缩写。

Linux操作系统包含了Linux内核与其他自由软件项目中的GNU组件和软件，可以成为GNU/Linux。

现在说的Linux操作系统是linux内核加软件。

总的来说最早的操作系统是unix，它非常的棒，商用版本改进后就成了macOS，另一部分伟大的人觉得这么好用的东西应该由全人类共享，于是根据unix设计了GNU/Linux。Linux可以说是开源项目中最伟大的项目了。使用了linux内核的都称作Linux操作系统。

树状文件结构FHS

了解过Linux有一个事情会让人印象深刻，那就是它的树状文件结构，从/开始，下面各个文件按照一定的规则存放文件。然后在这几年的科研过程中也接触到了非常多的软件，以及有些好文章的code里面都是遵照一定的规则存放代码。

这还是真是有讲究的，这种存放规则叫做：FHS（Filesystem Hierarchy Standard）是一个在Linux系统上的文件组织结构的标准。它规定了不同目录应该存放哪些类型的文件和它们的用途。主要是为了提供一个统一的文件系统布局，使得软件开发者和系统管理员能够在不同的Linux系统上更容易地理解每个文件夹的作用。

FHS针对目录树架构仅定义出三个目录下应该放置哪些数据，分别是下面三个目录
/（根目录）： 根目录是整个文件系统的起点，与开机系统有关。在根目录下存放着其它所有的目录和文件。 一些重要的子目录包括 /bin、/etc、/home、/usr、/var等，存放着不同类型的文件和目录。
/usr（unix software resource）： 此目录与软件安装和执行相关。它是用来存放用户和共享的只读数据，包括程序文件、文档和库文件等。 在/usr目录下，通常包含了一系列子目录： 如/usr/bin存放可执行的用户命令，/usr/lib存放共享库文件，/usr/share存放共享数据，/usr/include存放头文件等。

一切从根目录 (/) 开始

　　其实Linux启动时先把内核装载到内存中，然后再由内核去扫描激活树结构目录（这里就是 / 根目录）；根目录是整个系统最重要的一个目录，因为所有的目录都是由根目录衍生出来的，同时根目录也与开机、还原和系统修复等动作有关。由于系统开机时需要特定的开机软件、核心文件、开机所需程序、函式库等等文件数据，若系统出现错误时，根目录也必须要包含有能够修复文件系统的程序才行。因为根目录是这么的重要，所以在FHS的要求方面，它希望根目录不要放在非常大的分区，因为越大的分区内你会放入越多的数据，如此一来根目录所在分区就可能会有较多发生错误的机会。因此FHS标准建议：根目录(/)所在分区应该越小越好，且应用程序所安装的软件最好不要与根目录放在同一个分区内，保持根目录越小越好，这就可以保证根目录所在的文件系统也较不容易发生问题。说白了就是根目录和Windows的C盘一个样。

注：因为根目录与开机有关，开机过程中仅有根目录会被挂载，其它分区则是在开机完成之后才会持续的进行挂载。因此根目录下与开机过程有关的目录，就不能够与根目录放到不同的分区去。

补充：内核如何再开机时被加载到内存

　　其实在开机启动的过程是由引导加载程序（Boot Loader）负责的。引导加载程序是位于硬盘的引导扇区（通常为第一个物理扇区）中的一段代码，它会在系统加电启动时首先加载到内存，并负责加载操作系统内核。最常见的Linux引导加载程序是GRUB（GRand Unified Bootloader），它是用于多操作系统的通用引导加载程序。当计算机启动时，BIOS或UEFI会根据设定的启动顺序选择引导设备，并读取该设备的引导扇区。如果使用的是GRUB，那么引导扇区中的GRUB代码就会被加载到内存中。GRUB会提供一个菜单，列出可用的操作系统和内核映像。用户可以选择要启动的操作系统或内核。然后GRUB会根据用户的选择将相应的内核映像加载到内存中，并通过指定的命令行参数启动内核。因此，可以说Linux内核的启动是通过引导加载程序（如GRUB）来完成的。引导加载程序负责加载内核并传递控制权给内核，从而启动整个 Linux 系统。

一些目录文件基本说明
bin: 包含一些基本的命令二进制文件（可执行文件），这些文件通常可由所有用户执行。

lib: 包含一些基本的共享库和内核模块，这些库和模块供其它程序使用。
media: 是可移动媒体的挂载点，当插入可移动存储介质（如光盘、USB驱动器）时，这个目录会自动挂载。
mnt: 是一个用于临时挂载文件系统的目录。可以将其它文件系统（如移动硬盘、网络文件系统）挂载到这个目录下。
opt: 用于安装额外的应用程序软件包的目录，这些软件包通常不是系统自带的，而是后来添加的。
run: 存放与当前正在运行的进程相关的数据，如进程ID文件等。
sbin: 含有一些基本的系统二进制文件，用于系统管理和维护任务，只有系统管理员才能执行。
srv: 包含由系统提供的服务所需的数据，例如Web服务器的网页文件。
tmp: 用于存放临时文件的目录，这些文件在系统重启时可能会被清理掉（30天未查看修改则清理）。
usr: 是一个次要的目录层次结构，它类似于根目录（/），包含用户安装的应用程序、库文件、文档和其它非系统关键文件。
var: 包含一些变量数据，如日志文件、邮件、缓存和数据库等，这些数据可能会随着系统的运行而发生变化。

研究理念

我的研究理念一直受到双重因素的驱动 设计有效的模型和算法以解决现实世界的愿望 在此过程中提炼新数学问题时的挑战。在 本质，我的目标是为实际应用和进步做出贡献 数学本身。这种二元论是我 研究工作。

重点领域

我的主要研究领域集中在生物医学成像上，特别是在 临床图像重建、图像分析与辅助决策 利用图像和其他医疗数据进行诊断。

早期职业生涯

在我的博士和博士后期间，我开发了各种 偏微分方程框架内的模型和算法 （偏微分方程）和小波。其中包括设计脑动脉瘤的偏微分方程模型 分割以方便后续几何测量（参考文献）; 将偏微分方程、水平集方法和小波变换的概念集成到 创建用于心血管斑块定量的计算工具（参考文献）; 利用稀疏特性构建CT图像重建模型 小波帧变换的近似（Ref1、Ref2、Ref3）; 以及从数学上优化癌症放射治疗（参考文献） 后来由 phys.org 报道。

揭开 连接：偏微分方程和小波

在解决这些应用问题的过程中，我成为 人们越来越意识到偏微分方程和小波之间惊人的算法相似性。 这导致了与合作者的调查之旅，以探索任何 这两种范式之间的内在关系，已经演变 在成像领域独立工作近三十年。我们发现 深厚的联系，颠覆传统观点，建立起 基于小波的优化模型之间的渐近收敛关系 和偏微分方程模型。（参考文献1、参考文献2、参考文献3、参考文献4） 这种理论见解有效地弥合了 具有稀疏近似和多尺度性质的微分算子 小波，激发了融合了偏微分方程和 小波。特别是，这种洞察力导致了新的基于小波帧的图像 医学图像的分割模型（参考文献）。 值得注意的是，我们关于基于小波帧的分割的合作论文 超声视频（参考） 与我在上海交通大学和国立大学的同事一起 新加坡获得了媒体的关注，包括SIAM的报道 新闻， 科学 每日等

旅程 机器学习

我最初涉足机器学习的尝试始于 2014 年，结果是 将小波帧引入半监督的唯一论文 基于图和点云的学习模型（参考文献）， 以及与我的同事一起进行稀疏线性判别分析 来自亚利桑那大学的统计学项目（参考文献）。 然而，直到 2015-2016 年，我才完全掌握了革命性的 深度学习的影响，特别是对生物医学成像问题的影响。它是 也是在这段时间里，我们早期的工作揭示了这些联系 偏微分方程和小波之间促成了对这些的统一看法 数学模型，后来为我们在数学上理解提供了信息 深度神经网络。

随后，我带领我的团队调整了我们的重点 迈向深度学习方法。核心动机是探索 传统数学方法（如偏微分方程）和 深度学习中的小波和数据驱动模型。这次探索 在2017年至2018年期间，我的两件重要作品达到了顶峰： 数值微分方程与深层微分方程之间联系的建立 神经网络架构，导致 ODE-Net 的发展（参考文献） 和 PDE-Net（参考文献 1、[参考文献 2](http://bicmr.pku.edu.cn/~dongbin/Publications/PDE-Net 2.0.pdf)）、 在大规模数据集上进行了实证验证。通过这个和 在随后的几次研究中，我们逐渐开发出一种新的仿制药 协同整合数学建模和机器的方法 学习技巧。 我们一直在不断练习和 从那时起，在我的研究工作中完善了这种通用方法（例如 “绕道科学计算”中的例子）。

我在 2022 年国际大会上演讲的核心焦点 数学家（ICM）将阐明偏微分方程之间错综复杂的联系 和小波，以及离散微分方程和 深度神经网络。这些“桥梁”对计算成像和 科学计算，在我的演讲中进行了深入的讨论，并且 相应的会议记录和注释（参考文献1，参考文献2）。

涉足科学计算领域

通过数值视角理解深度神经网络 微分方程，解决科学挑战变得很自然 计算，特别是基于偏微分方程的正向和逆问题。对于 向前问题，我们混合了守恒定律的数值方法和 具有深度强化学习和元学习的多网格方法。（参考文献 1、参考文献 2）最近，我们发现 元学习和降阶建模之间的联系，铺平了道路 一种逼近参数偏微分方程解流形的新技术。 （参考文献 1、参考文献 2）对于逆问题，我们的 重点主要放在成像上，我们提出了几种融合方法 使用传统数学算法（Ref1、Ref2、Ref3、Ref4）的深度学习技术。

临床 应用和转化研究

自2019年以来，我开始了与北京的广泛合作 大学肿瘤医院，专注于临床辅助诊断问题。 这导致了医学图像分析的一系列令人振奋的进展， 为临床实践提供有价值的工具，这些工具已经取得了成功 采用和一定程度的转化实施。（参考文献1、参考文献2、参考文献3、参考文献4）

未来研究 途径

在我的学术轨迹中，我遇到了两个主要挑战： 将指导我未来 n+1 年的研究重点。

― 人工智能 计算成像：解决生物医学成像的“精品”性质 模型：

第一个挑战的根源在于需要一个统一而强大的模型 在生物医学成像领域。目前，虽然融合了机器 使用第一性原理建模进行学习已被证明是强大的，其实用性通常 依赖于为特定场景设计特定模型 既不有效也不普遍的“精品”方法 适用。有鉴于此，我未来的研究旨在构建一个 计算成像的基础模型。这项工作将集中在两个方面 关键任务：首先，将计算的三个阶段无缝集成 成像，即传感、重建和分析;其次，要 将各种成像模式合并到一个综合框架中。这 统一模型旨在更好地应对在现实世界中遇到的挑战 生物医学成像。

在计算成像中寻找基础模型是 与支配波物质的普遍原则有着内在的联系 相互作用，几乎是所有成像模式的基础。这些 相互作用可以通过偏微分方程优雅地描述。认识到这一点，我们的 研究努力为基础模型奠定基础 包括成像过程中固有的各种偏微分方程（和 超越）。我们目前的发展轨迹使我们在以下方面取得了可喜的进展： 一维域 （Ref）， 标记 实现整体模型的全面旅程的初始阶段 用于计算成像。

― 人工智能 数学：构建用于理论研究的 AI Copilot

第二个挑战源于对 人工智能技术已经超越了底层的发展 理论。传统的理论研究在指导 算法的设计，但它在人工智能领域越来越落后。一 关键瓶颈是理论驱动研究的低效率，研究人员 经常发现很难灵活地应用广泛的数学知识 或创新地创造新的数学概念和工具。为了解决这个问题， 我未来的工作计划是开发一个人工智能助手，旨在促进 数学理论研究。该助手将具有双重功能： 激发数学思维，实现理论的多个方面的自动化 工作。这包括但不限于定理证明的验证， 自动定理证明和数学概念的语义搜索和 结果。虽然我们已经朝着这个方向迈出了一些初步（但令人兴奋）的步骤 与香港大学的同事合作，特别是在 应用机器学习工具探索仿射Deligne-Lusztig变种（参考文献），仍然存在 要充分发挥人工智能作为推动进步工具的潜力，还有很多工作要做 数学理论。

通过关注这两个关键挑战，我的目标是 为生物医学成像领域和 数学本身的进步。

获得的一些启发

数学和计算机深度学习交叉进行建模依然会有广阔的空间。建模型总要去和实际问题匹配，现在生物信息学领域仍有大量的复杂的生物机制建模问题可以去探索。

我们探索的求解CME问题非常有意义。不论是利用更新的神经网络求解CME，还是建一个更复杂的CME模型。它们都是宏观的，不具体的。纵向深入下去，就去研究一个具体的实例去研究透将会更有意思。就像整个基因相关的研究中，不管建怎么样的模型，利用多么先进的数据，最终都是要找到那几个关键的基因。

  1. 所以第一步我们的第一个实例研究什么内容呢？我个人觉得还是去研究基因间的调控关系，这方面有大量的成熟方法可以做参考，可以验证我们的第一步是不是对的。这一步如果可行就做的快一点，这里面从Gene Regulatory Network–>Biochemical Reaction Network–>Deep Neural Network，这里面的三个网络能否合并存储或进行相关的优化提高运行效率呢？

  2. 第二步将单细胞里面其他的生物过程也建模出来(可以参照现有的方法去做，不一定非要都建模成CME)，为下一步一个统一的大模型做好前期准备。这一步可以参考上一步的经验，需要大量的工作。1、先梳理清楚单细胞内哪些生物过程是必须考虑的，找相关的综述相互验证确定。这里也要对应考虑现已经开发出来的单细胞数据有哪些反推其对应的生物过程。2、将每一个生物过程进行建模，能建模成CME就建模成CME，不能也可以依据这个领域的文献建模成ODE、PDE等等。3、一定要求解得到结果，不管是模拟结果（数值解）还是解析解。然后就是和实验结果进行比较进行验证，让各个生物过程都是有实验依据有说服力的。

  3. 第三步就是建模一个单细胞的CME模型，这个模型能反映该细胞的命运变化。即细胞核中基因如何控制细胞内的生物过程，然后一步步演化开启了细胞的下一个阶段，这里面相对于之前要考虑细胞的周期演化。进一步的要考虑细胞接收的外部信号如何添加到模型中并如何去影响细胞命运演变。最后由于CME模式是随机的模型，相应也要将该单细胞放在细胞簇(群体)中去考虑，看大趋势上细胞们是如何去选择自己的命运。

一些不成熟的想法欢迎指导批评和联系邮箱交流。

5. 优点和不足

优点：完全免费；静态站点，轻量快速；可按需求自由定制改造；托管在 GitHub，安全省心；迁移方便…

不足：发文不便，依赖于本地环境；更适合个人博客使用；GitHub 在国内访问速度有点不快。

三、Next 主题和其他资源

6. 使用 Next 主题美化个人博客

Hexo 默认的 Landscape 主题不够美观，换成流行的 Next 主题吧！通过对 Next 主题简单的设置，基本可以满足新手所有的需求。

7. 其他资源

除 Next 之外，Hexo 还有非常丰富的主题，可以浏览 Hexo 官网寻找适合自己口味的主题。

随着博客数量和网站需求的增加，可能需要多种多样的扩展功能，可以在 Hexo 官网插件页面找到满足自己需求的插件。

四、绑定域名（可选）

这里也可以免费使用作者的二级域名形如a.nyada.cn（a为自定义域名区，限定前十个联系的人）。博客搭建完成使用的是 GitHub 的子域名（如用户名.github.io），我们可以为 Hexo 博客绑定自己的域名替换 GitHub 域名，更加个性化和专业，也利于 SEO。

8. 域名注册和解析

利用域名注册网站，如阿里云、腾讯云等。然后在 DNS 设置部分，删除自带的记录，然后添加 CNAME 记录将 www 域名解析指向：用户名.github.io。

9. 绑定域名到 Hexo 博客

进入本地博客文件夹的 source 目录，打开记事本，里面输入自己的域名，如 http://a.nyada.cn，保存名称为 “CNAME”，格式为 “所有文件”（无 .txt 后缀）。

10. 开启 HTTPS

配置自己的域名后，需要我们手动开启 HTTPS。打开博客所在 GitHub 仓库，Settings -> 下拉找到 GitHub Pages -> 勾选 Enforce HTTPS。

end

知乎crystal总结说：

Hexo 是一种纯静态的博客，我们必须要在本地完成文章的编辑再部署到 GitHub 上，依赖于本地环境。不能像 WordPress 或 Typecho 那样的动态博客一样能直接在浏览器中完成撰文和发布。

可以说是一种比较极客的写博客方式，但是优势也是明显的——免费稳定省心，比较适合爱折腾研究的用户，或者没有在线发文需求的朋友。

categories

Posted on 2024-04-02 | In a

单词解释

“categories” 是英文中的一个词，通常用于描述事物或概念的分类或类别。在计算机领域，尤其是在网站或博客的开发中，”categories” 通常指代文章或内容所属的类别或分类。在网站或博客的内容管理系统中，可以使用 “categories” 来将文章或内容进行分类，以便读者更容易地找到感兴趣的内容。在中文中，”categories” 可以翻译为 “分类” 或 “类别”。

categories和tags的区别

“categories”（分类）和”tags”（标签）在内容管理系统中用于对文章或内容进行分类和标记，但它们在功能和用途上有一些区别：

  1. 分类（Categories）：
    • 分类通常用于对文章或内容进行更广泛的主题性分类。它们代表了文章所属的主要类别或主题。例如，一个博客可能有 “科技”、”健康”、”旅行” 等分类。
    • 一篇文章通常只能属于一个分类，它代表了文章的主要内容或主题。
  2. 标签（Tags）：
    • 标签则是用于对文章或内容进行更具体的描述性标记。标签通常用于描述文章的关键词、主题或内容特点，使读者能够更容易地找到相关内容。例如，对于一篇关于健康的文章，可能会有 “健康”、”饮食”、”运动” 等标签。
    • 一篇文章可以有多个标签，这样可以更细致地描述文章的内容特点。

因此，分类更多地用于对文章进行宏观的整体分类，而标签则更适合用于对文章进行细致的描述和关联。

如何在网站中设置categories

在网站中设置文章分类（categories）通常是通过网站的内容管理系统（CMS）或使用特定的网站建设平台来完成的。下面是一般的步骤：

  1. 登录后台管理界面： 使用你的管理员账户登录到你的网站后台管理界面。
  2. 找到分类设置选项： 寻找一个用于管理文章分类的选项或页面。这通常在网站管理界面的“文章”或“内容”部分。
  3. 创建分类： 一般情况下，你会看到一个可以添加、编辑或删除文章分类的界面。在这里，你可以创建新的分类，例如：“科技”、“健康”、“旅行”等。
  4. 指定文章分类： 在撰写或编辑文章时，你应该能够从已创建的分类中选择一个适当的分类。这通常在撰写文章的编辑界面中的一个下拉菜单或复选框中完成。
  5. 保存更改： 选择好文章分类后，确保保存你的文章或更改。这样，文章就会被归类到你所选择的分类中。

具体的步骤可能会因网站建设平台或CMS系统而异，但通常都遵循类似的流程。如果你使用的是特定的网站建设平台（如WordPress、Wix、Squarespace等），可以参考相关平台的帮助文档或社区支持资源，了解更详细的指导。
